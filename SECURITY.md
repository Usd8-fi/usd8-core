# Security Policy

## Reporting a Vulnerability

Please report security issues privately to **rick@usd8.fi**.

Do not open a public GitHub issue for suspected vulnerabilities. Include:

- a description of the issue and its impact,
- a proof-of-concept or steps to reproduce,
- the commit hash or deployment address affected,
- your preferred contact channel for follow-up.

We will acknowledge receipt within 72 hours and aim to provide a remediation
timeline within 7 days.

## Scope

- `src/USD8.sol` and any deployed instance of the `USD8` token.
- Deployment scripts under `script/`.

Out of scope: third-party libraries (report upstream to OpenZeppelin / Foundry),
front-ends not maintained in this repo.

## Disclosure

We follow coordinated disclosure. We request that researchers delay public
disclosure until a fix is deployed and users have had reasonable time to
upgrade or take protective action.

## Timing

This project is under heavy development and not ready yet. Please wait till the project is finished before submitting reports.
