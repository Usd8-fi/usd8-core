## Required setup

- Ethereum mainnet RPC URL in `ETH_RPC_URL`.
- A securely configured Foundry broadcaster (keystore, hardware wallet, or another supported signer). Do not put a raw private key in this repository.
- Foundry installed and dependencies available (`forge install`).
- `ETHERSCAN_API_KEY` only when using `--verify`.
- The step-01 timelock address for step 02.
- The canonical external contracts hardcoded in step 02 must contain the expected mainnet code: Morpho Vault V2 factory, USD8 booster, wstETH, oracle feeds, and the configured Aave/Morpho USDC vaults.
- After reviewing the launch Aave strategy and vault, set `AAVE_STRATEGY_REVIEWED=true`; step 02 fails closed without this explicit confirmation.

Before production deployment, review every hardcoded address and parameter in both scripts. In particular, `AAVE_USDC_VAULT` is currently marked in the source as needing an update before live deployment.

## Governance requirements

Step 01 creates a timelock with:

- 24-hour minimum delay;
- `DEFAULT_PROPOSER` as sole proposer and canceller;
- open execution (`address(0)` executor);
- the timelock as its own admin, with no external admin.

Step 02 rejects a timelock that does not have that exact delay and role configuration. `DEFAULT_ADMIN` in step 02 must match the proposer/canceller configured by step 01. Step 02 performs bootstrap configuration using temporary deployer authority, then transfers final Registry and vault governance to the validated timelock.

## Deploy

```bash
source .env

# Step 1: deploy the governance timelock.
forge script script/01_DeployTimelock.s.sol:DeployTimelockScript \
  --rpc-url "$ETH_RPC_URL" --broadcast --verify \
  --etherscan-api-key "$ETHERSCAN_API_KEY"

# Copy the address printed by step 1.
TIMELOCK_ADDRESS=0x...

# Step 2: deploy, seed, wire, and hand off the complete system.
forge script script/02_DeployUSD8System.s.sol:DeployUSD8SystemScript \
  --sig "run(address)" "$TIMELOCK_ADDRESS" \
  --rpc-url "$ETH_RPC_URL" --broadcast --verify \
  --etherscan-api-key "$ETHERSCAN_API_KEY"
```

Use the same commands without `--broadcast` for simulation. Add the signer option appropriate for the configured Foundry account.

## Failure and recovery

A Foundry script broadcasts multiple transactions; the complete deployment is not one atomic transaction. If broadcasting succeeds partially or verification is interrupted, preserve the broadcast artifacts and use `--resume`. Do not blindly rerun the deployment, because already-created contracts and the permanent seed transfer are not rolled back.

Before accepting the deployment, verify the logged addresses, timelock roles and delay, Registry timelock/admin state, beacon ownership, canonical Registry pointers, sUSD8 seed balance, scoring configuration, insured-token configuration, and Treasury profit receivers.
